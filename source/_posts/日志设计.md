# 日志设计

日志是软件开发中不可或缺的重要组成部分，它记录了程序的运行状态、错误信息和调试信息。一个良好的日志系统可以帮助开发者快速定位问题、监控系统运行状况和分析用户行为。本文将介绍日志设计的基本原则和多种编程语言的实现方式。

## 日志级别

日志级别是用来区分日志信息重要程度的标准。常见的日志级别从高到低依次为：

| 级别 | 数值 | 说明 | 使用场景 |
|------|------|------|----------|
| OFF | 0 | 关闭所有日志 | 生产环境关闭日志 |
| FATAL | 1 | 致命错误 | 导致程序无法继续运行的严重错误 |
| ERROR | 2 | 错误 | 程序可以继续运行但出现的问题 |
| WARN | 3 | 警告 | 不影响运行但可能存在问题的情况 |
| INFO | 4 | 信息 | 重要的业务节点和程序状态 |
| DEBUG | 5 | 调试 | 开发阶段使用的详细信息 |
| TRACE | 6 | 跟踪 | 最详细的日志，用于问题诊断 |

### 日志级别的选择原则

1. **FATAL**：程序崩溃、数据库连接失败等无法恢复的错误
2. **ERROR**：文件操作失败、网络超时等影响功能的问题
3. **WARN**：配置缺失、性能下降等需要注意的情况
4. **INFO**：服务启动/停止、用户登录、重要业务操作
5. **DEBUG**：函数入口/出口、变量值变化、中间状态
6. **TRACE**：详细到每一行代码的执行轨迹

## 日志设计原则

### 1. 日志内容原则

- **有意义的描述**：日志信息应该清晰描述发生的情况
- **包含上下文**：记录足够的上下文信息便于问题定位
- **避免敏感信息**：不要记录密码、密钥等敏感数据
- **适当的详细程度**：避免过多无用的日志，也不要遗漏关键信息

### 2. 日志性能原则

- **异步日志**：高并发场景使用异步写入避免阻塞
- **日志分级**：生产环境关闭DEBUG/TRACE级别
- **滚动日志**：避免单文件过大

### 3. 日志格式原则

标准的日志格式应包含：

```
[时间戳] [级别] [线程ID] [模块名] [函数名:行号] 消息内容
```

# C语言版本

C语言没有内置的日志库，通常需要通过宏定义和函数来实现日志功能。

## 基础版本

```c
/**********************************************
*                LOG DEFINE                   *
**********************************************/
// 日志级别定义
// OFF FATAL ERROR WARN INFO DEBUG TRACE
//  0    1     2     3    4    5     6
#define LOG_LEVEL_OFF   (0)
#define LOG_LEVEL_FATAL (1)
#define LOG_LEVEL_ERROR (2)
#define LOG_LEVEL_WARN  (3)
#define LOG_LEVEL_INFO  (4)
#define LOG_LEVEL_DEBUG (5)
#define LOG_LEVEL_TRACE (6)

// 设置日志级别（DEBUG以上会输出）
#define LOG_LEVEL LOG_LEVEL_DEBUG

// 日志宏定义
#define LOG(level, format, ...) \
    if(LOG_LEVEL >= level) { \
        fprintf(stderr, "[%s|%s@%s,%d] " format "\n", \
            #level, __func__, __FILE__, __LINE__, ##__VA_ARGS__ ); \
    }

// 便捷宏定义
#define LOG_FATAL(format, ...)   LOG(LOG_LEVEL_FATAL, format, ##__VA_ARGS__)
#define LOG_ERROR(format, ...)   LOG(LOG_LEVEL_ERROR, format, ##__VA_ARGS__)
#define LOG_WARN(format, ...)    LOG(LOG_LEVEL_WARN, format, ##__VA_ARGS__)
#define LOG_INFO(format, ...)    LOG(LOG_LEVEL_INFO, format, ##__VA_ARGS__)
#define LOG_DEBUG(format, ...)   LOG(LOG_LEVEL_DEBUG, format, ##__VA_ARGS__)
#define LOG_TRACE(format, ...)   LOG(LOG_LEVEL_TRACE, format, ##__VA_ARGS__)
```

## 增强版本：带文件输出

```c
#ifndef LOG_H
#define LOG_H

#include <stdio.h>
#include <time.h>
#include <string.h>

typedef enum {
    LOG_LEVEL_OFF = 0,
    LOG_LEVEL_FATAL,
    LOG_LEVEL_ERROR,
    LOG_LEVEL_WARN,
    LOG_LEVEL_INFO,
    LOG_LEVEL_DEBUG,
    LOG_LEVEL_TRACE
} LogLevel;

// 日志级别名称
static const char* LOG_LEVEL_NAMES[] = {
    "OFF", "FATAL", "ERROR", "WARN", "INFO", "DEBUG", "TRACE"
};

// 全局日志级别
extern LogLevel g_log_level;

// 设置日志级别
void log_set_level(LogLevel level);

// 初始化日志文件
int log_init(const char* filename);

// 关闭日志
void log_close();

// 日志函数
void log_message(LogLevel level, const char* file, int line,
                 const char* func, const char* format, ...);

// 宏定义
#define LOG(level, format, ...) \
    log_message(level, __FILE__, __LINE__, __func__, format, ##__VA_ARGS__)

#define LOG_FATAL(format, ...)   LOG(LOG_LEVEL_FATAL, format, ##__VA_ARGS__)
#define LOG_ERROR(format, ...)   LOG(LOG_LEVEL_ERROR, format, ##__VA_ARGS__)
#define LOG_WARN(format, ...)    LOG(LOG_LEVEL_WARN, format, ##__VA_ARGS__)
#define LOG_INFO(format, ...)    LOG(LOG_LEVEL_INFO, format, ##__VA_ARGS__)
#define LOG_DEBUG(format, ...)   LOG(LOG_LEVEL_DEBUG, format, ##__VA_ARGS__)
#define LOG_TRACE(format, ...)   LOG(LOG_LEVEL_TRACE, format, ##__VA_ARGS__)

#endif // LOG_H
```

## C语言版本实现

```c
#include "log.h"
#include <stdarg.h>
#include <stdlib.h>

static FILE* g_log_file = NULL;
static LogLevel g_log_level = LOG_LEVEL_INFO;

void log_set_level(LogLevel level) {
    g_log_level = level;
}

int log_init(const char* filename) {
    if (filename == NULL) {
        g_log_file = stderr;
        return 0;
    }

    g_log_file = fopen(filename, "a");
    if (g_log_file == NULL) {
        return -1;
    }
    return 0;
}

void log_close() {
    if (g_log_file != NULL && g_log_file != stderr) {
        fclose(g_log_file);
        g_log_file = NULL;
    }
}

void log_message(LogLevel level, const char* file, int line,
                 const char* func, const char* format, ...) {
    if (level > g_log_level) {
        return;
    }

    // 获取当前时间
    time_t now = time(NULL);
    struct tm* tm_now = localtime(&now);
    char time_str[32];
    strftime(time_str, sizeof(time_str), "%Y-%m-%d %H:%M:%S", tm_now);

    // 写入日志
    fprintf(g_log_file, "[%s] [%s] [%s:%d %s] ",
            time_str, LOG_LEVEL_NAMES[level], file, line, func);

    // 处理可变参数
    va_list args;
    va_start(args, format);
    vfprintf(g_log_file, format, args);
    va_end(args);

    fprintf(g_log_file, "\n");

    // 确保写入
    fflush(g_log_file);
}
```

## 使用示例

```c
#include "log.h"

int main() {
    // 初始化日志，输出到文件（NULL输出到stderr）
    log_init("app.log");

    // 设置日志级别
    log_set_level(LOG_LEVEL_DEBUG);

    LOG_INFO("应用程序启动");
    LOG_DEBUG("配置加载完成，共 %d 个配置项", 10);
    LOG_WARN("配置文件不存在，使用默认配置");
    LOG_ERROR("数据库连接失败: %s", "Connection refused");
    LOG_FATAL("严重错误，程序即将退出");

    log_close();
    return 0;
}
```

# Python版本

Python中有多种日志库，最常用的是内置的`logging`模块。

## 基础配置

```python
import logging
import logging.handlers
import os

class Log:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._initialized = False
        return cls._instance

    def __init__(self):
        if self._initialized:
            return

        self.logger = logging.getLogger('app')
        self.logger.setLevel(logging.DEBUG)

        # 控制台处理器
        console_handler = logging.StreamHandler()
        console_handler.setLevel(logging.INFO)
        console_formatter = logging.Formatter(
            '[%(asctime)s] [%(levelname)s] [%(name)s] %(message)s',
            datefmt='%Y-%m-%d %H:%M:%S'
        )
        console_handler.setFormatter(console_formatter)

        # 文件处理器（滚动）
        file_handler = logging.handlers.RotatingFileHandler(
            'app.log',
            maxBytes=10*1024*1024,  # 10MB
            backupCount=5,
            encoding='utf-8'
        )
        file_handler.setLevel(logging.DEBUG)
        file_formatter = logging.Formatter(
            '[%(asctime)s] [%(levelname)s] [%(filename)s:%(lineno)d %(funcName)s] %(message)s',
            datefmt='%Y-%m-%d %H:%M:%S'
        )
        file_handler.setFormatter(file_formatter)

        self.logger.addHandler(console_handler)
        self.logger.addHandler(file_handler)
        self._initialized = True

    def debug(self, msg, *args, **kwargs):
        self.logger.debug(msg, *args, **kwargs)

    def info(self, msg, *args, **kwargs):
        self.logger.info(msg, *args, **kwargs)

    def warning(self, msg, *args, **kwargs):
        self.logger.warning(msg, *args, **kwargs)

    def error(self, msg, *args, **kwargs):
        self.logger.error(msg, *args, **kwargs)

    def critical(self, msg, *args, **kwargs):
        self.logger.critical(msg, *args, **kwargs)

# 全局日志实例
log = Log()

# 使用示例
if __name__ == '__main__':
    log.info('应用程序启动')
    log.debug('配置加载完成，共 %d 个配置项', 10)
    log.warning('配置文件不存在，使用默认配置')
    log.error('数据库连接失败: %s', 'Connection refused')
    log.critical('严重错误，程序即将退出')
```

# Java版本

Java中有SLF4J作为日志门面，配合Logback或Log4j2作为实现。

## Maven依赖

```xml
<dependencies>
    <dependency>
        <groupId>org.slf4j</groupId>
        <artifactId>slf4j-api</artifactId>
        <version>1.7.36</version>
    </dependency>
    <dependency>
        <groupId>ch.qos.logback</groupId>
        <artifactId>logback-classic</artifactId>
        <version>1.2.11</version>
    </dependency>
</dependencies>
```

## SLF4J使用示例

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class UserService {
    // 使用类的全限定名作为logger name
    private static final Logger logger = LoggerFactory.getLogger(UserService.class);

    public void login(String username) {
        logger.info("用户登录: username={}", username);

        try {
            // 业务逻辑
            validateUser(username);
            logger.debug("用户验证通过: username={}", username);
        } catch (Exception e) {
            // 使用占位符避免字符串拼接开销
            logger.error("用户登录失败: username={}, error={}", username, e.getMessage());
            throw e;
        }
    }
}
```

## Logback配置

```xml
<!-- logback.xml -->
<configuration>
    <property name="LOG_PATTERN" value="%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"/>

    <!-- 控制台输出 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>${LOG_PATTERN}</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 文件输出 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>app.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>app.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>${LOG_PATTERN}</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 异步输出 -->
    <appender name="ASYNC_FILE" class="ch.qos.logback.classic.AsyncAppender">
        <appender-ref ref="FILE"/>
        <queueSize>512</queueSize>
        <discardingThreshold>0</discardingThreshold>
    </appender>

    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="ASYNC_FILE"/>
    </root>

    <logger name="com.example" level="DEBUG"/>
</configuration>
```

# Go版本

Go语言中常用的日志库有`logrus`、`zap`等。

## 使用logrus

```go
package main

import (
    "os"
    "github.com/sirupsen/logrus"
)

var log *logrus.Logger

func init() {
    log = logrus.New()

    // 输出到文件
    file, err := os.OpenFile("app.log", os.O_CREATE|os.O_WRONLY|os.O_APPEND, 0666)
    if err == nil {
        log.SetOutput(file)
    }

    // 设置格式
    log.SetFormatter(&logrus.JSONFormatter{
        TimestampFormat: "2006-01-02 15:04:05",
    })

    // 设置级别
    log.SetLevel(logrus.DebugLevel)
}

func main() {
    log.WithFields(logrus.Fields{
        "username": "zhangsan",
        "action":   "login",
    }).Info("用户登录")

    log.WithFields(logrus.Fields{
        "error": "connection refused",
    }).Error("数据库连接失败")
}
```

## 使用zap（高性能）

```go
package main

import (
    "go.uber.org/zap"
)

var logger *zap.Logger

func init() {
    // 开发模式
    cfg := zap.NewDevelopmentConfig()
    cfg.OutputPaths = []string{"app.log"}
    logger, _ = cfg.Build()
    defer logger.Sync()
}

func main() {
    // 结构性日志
    logger.Info("用户登录",
        zap.String("username", "zhangsan"),
        zap.Int("user_id", 12345),
    )

    // 使用SugarLogger（性能略低但支持格式化）
    sugar := logger.Sugar()
    sugar.Infof("用户 %s 登录成功", "zhangsan")
    sugar.Errorf("数据库连接失败: %v", "connection refused")
}
```

# 日志最佳实践

## 1. 不要记录敏感信息

```java
// 错误示例
logger.info("用户登录: username={}, password={}", username, password);

// 正确示例
logger.info("用户登录: username={}", username);
```

## 2. 使用占位符而非字符串拼接

```java
// 错误示例
logger.info("用户信息: " + user.toString());

// 正确示例
logger.info("用户信息: username={}, age={}", user.getUsername(), user.getAge());
```

## 3. 记录结构化日志

```json
{
    "timestamp": "2021-01-02 20:41:22",
    "level": "INFO",
    "service": "user-service",
    "trace_id": "abc123",
    "message": "用户登录成功",
    "data": {
        "user_id": 12345,
        "username": "zhangsan"
    }
}
```

## 4. 日志级别使用建议

| 场景 | 级别 |
|------|------|
| 程序启动/关闭 | INFO |
| 配置加载 | INFO |
| 请求入口/出口 | DEBUG |
| 业务关键步骤 | INFO |
| 外部服务调用 | DEBUG |
| 参数验证失败 | WARN |
| 业务异常 | ERROR |
| 系统错误 | ERROR |
| 致命错误无法恢复 | FATAL |

## 5. 日志输出 Checklist

- [ ] 时间戳（精确到毫秒）
- [ ] 日志级别
- [ ] 线程ID（多线程环境）
- [ ] 模块名/类名
- [ ] 函数名和行号
- [ ] TraceID（分布式追踪）
- [ ] 关键变量值

# 总结

| 语言 | 推荐日志库 | 特点 |
|------|-----------|------|
| C | 自定义宏/log4c | 轻量、可定制 |
| Python | logging/loguru | 丰富、生态好 |
| Java | SLF4J+Logback | 门面模式、性能好 |
| Go | zap/logrus | 高性能、结构化 |
| C++ | spdlog | 头部文件、C++11 |

选择合适的日志方案，可以让问题定位更加高效，监控更加清晰。
